Cancellation¶

Схема¶

id:

uid, генерується автоматично
reason:

рядок, багатомовний, обов’язковий

Причина через яку скасовується закупівля.
status:

рядок

Можливі значення:
pending:

За замовчуванням. Запит оформляється.
active:

Скасування активовано.
documents:

Список об’єктів Document

Супровідна документація скасування: Протокол рішення Тендерного комітету Замовника про скасування закупівлі.
date:

рядок, Date

Дата скасування.
cancellationOf:

рядок, обов’язковий

Можливі значення та значення за замовчуванням:

  • tender
relatedLot:

рядок

ID пов’язаного об’єкта Lot.

Схема роботи Cancellation¶

digraph G { A [ label="pending*" ] B [ label="active"] A -> B; }

* позначає початковий стан
